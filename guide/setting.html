# 기본 설정

VuePress를 시작하기 위해 몇가지 설정을 진행합니다. 순차적으로 진행하세요.

NOTE

신규 프로젝트에 VuePress를 사용할 경우, 프로젝트 초기화를 먼저 수행한 후 VuePress를 설치합니다.

 


$ yarn init
# npm init
1
2

# 문서 디렉토리

Markdown 문서를 보관 관리 할 디렉토리 docs를 프로젝트 루트에 생성합니다.

$ mkdir docs
1

프로젝트 디렉토리 구조는 다음과 같습니다.



 




.
├── README.md
├── docs/
├── node_modules/
├── package.json
└── yarn.lock

# 홈페이지 문서

VuePress로 제작할 사이트의 홈페이지가 될 index.md 파일을 docs/ 디렉토리에 생성합니다.

$ echo '# VuePress 시작하기' > docs/index.md
1

NOTE

index.md 대신 Markdown 문서 작성 관습에 따라 README.md 파일을 사용할 수 있습니다.

# NPM 스크립트

VuePress 실행 명령을 단축하기 위해 package.json 파일에 스크립트 명령을 추가합니다.



 
 



{
  "scripts": {
    "dev": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}

# 문서 작성

NPM 스크립트 명령 dev를 실행한 후, Markdown 문서를 작성하면 실시간으로 md 파일이 html 파일로 변경되어 VuePress 사이트에 표시됩니다.

 


$ yarn dev
# npm run dev
1
2

NOTE

개발 모드를 끝내리면 Ctrl + C를 누릅니다.

# 사이트 배포

문서 작성이 마무리되면 사이트 배포를 위해 build 명령을 수행합니다.

 


$ yarn build
# npm run build
1
2

NOTE

기본 배포 위치는 .vuepress/dist 입니다. 배포 위치를 변경하려면 .vuepress/config.js 설정 파일을 만든 후, dest 속성 값으로 배포 위치를 설정합니다.




 


// docs/.vuepress/config.js

module.exports = {
  dest: 'build'
};
